# QA Checklist

## 必过项

- 是 16:9 横版。
- 背景是干净白底。
- 有夜航猫头鹰。
- 夜航猫头鹰承担核心动作，不只是装饰。
- 猫头鹰不是黑色实心剪影。
- 猫头鹰保留暖橙脸盘/耳羽、蓝绿色身体、黑墨镜、深蓝外袍、白咖啡杯这些核心识别点。
- 内容隐喻来自正文，而不是来自 NightSailing、夜航船或角色设定。
- 没有把 NightSailing、夜航、夜航船写到图里。
- 没有默认加入船、帆、海浪、码头、港口、灯塔、月亮、星空、航线、船锚、舷窗等元素。
- 默认生成时没有复刻旧案例构图，而是为当前文章生成新隐喻。
- 用户要求重构旧图时，只替换主角，保留原图构图、标签、箭头、物件和核心动作。
- 画面怪诞、有创意、有意思。
- 简洁清爽，主体不超过画面约 60%。
- 一张图只讲一个核心结构。
- 中文标注少、短、能读。
- 橙色只用于主路径或箭头。
- 红色只用于重点、问题、提醒或结果。
- 蓝色只用于补充说明、反馈或系统状态，或角色固定色。

## 失败信号

出现以下情况，重生成或局部编辑：

- 猫头鹰被画成黑色实心角色。
- 猫头鹰像黑色实心角色戴了猫头鹰耳朵。
- 猫头鹰只是站在旁边，没有参与核心动作。
- 猫头鹰过大、过复杂，抢走结构表达。
- 羽毛细节、衣服褶皱或咖啡杯细节太多。
- 图里出现 NightSailing、夜航、夜航船等品牌/世界观文字。
- 图里无故出现船、灯塔、海、码头、月亮、星空、航海符号。
- 左上角有“常见坑 / Workflow / 系统架构图 / 路线图”等标题。
- 画面像 PPT、课程课件、正式流程图。
- 元素太多、箭头太多、节点太多。
- 文字变成大段解释。
- 背景有纸纹、阴影、渐变、米色、噪点。
- 真实 UI 截图或科技感界面。
- 中文错字严重或标注不可读。
- 画面太可爱、幼稚、3D、商业 mascot。

## 迭代方法

- 太普通：让夜航猫头鹰成为动作主体，加入一个来自正文的奇怪但成立的隐喻。
- 夜航概念入侵：删除所有船、海、灯塔、夜景、星月和 NightSailing 文案，只保留猫头鹰主角。
- 太复杂：删节点，只保留一个动作和 3-5 个短标注。
- 角色太抢：缩小猫头鹰，减少羽毛线和衣褶，保留核心色块。
- 太简化：恢复 `assets/nightsailing-owl-reference.png` 的味道，保留墨镜、耳羽、脸盘、蓝袍和咖啡杯。
- 太可爱：强调 deadpan、blank serious expression、not cute、not mascot。
- 太 PPT：去掉标题、边框、整齐网格和过多箭头，改成手绘场景。
- 文字错：优先局部编辑；错得太多就重生成并减少标注数量。

## 交付判断

高质量图应该让读者先看到“这个彩色猫头鹰在认真干一件怪事”，然后 1 秒内看懂正文结构。读者不应该觉得这是一张 NightSailing 宣传图，也不应该在画面里看到与正文无关的船、灯塔或夜景符号。
